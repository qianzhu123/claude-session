MCP 与 Skill 管理

把常见下载、安装、查看、移除动作整理成可复制命令；实际执行前仍由终端确认。

M

查看 MCP 服务

确认当前项目已经连接的 MCP server。

claude mcp list
+

添加 MCP 服务

把本地或 npx/uvx 启动的服务注册到 Claude Code。

claude mcp add <name> -- <command> <args>
S

打开 Skill 目录

下载或整理本机 Claude Code skills。

explorer %USERPROFILE%\.claude\skills
↓

下载 Skill 模板

把仓库型 skill 放入本机 skills 目录后重启会话。

git clone <repo-url> "%USERPROFILE%\.claude\skills\<skill-name>"

常用操作

设置定时任务

Windows 使用任务计划程序运行固定目录下的 Claude 命令。

schtasks /Create /SC DAILY /TN "Claude Daily" /TR "cmd /c cd /d D:\path && claude --permission-mode plan "今日巡检"" /ST 09:00

多 agent 模式

把目标拆成互不共享状态的子任务，并在提示词中要求主 agent 汇总。

请用多 agent：A 负责代码检索，B 负责实现，C 负责验证，最后合并结论。

提示词骨架

目标：完成...
范围：只改...
权限：先计划，再执行；危险命令需确认
约束：遵循现有项目风格
验收：运行...，给出变更文件和结果

斜杠命令速查

命令会随 Claude Code 版本变化，最终以终端内 /help 输出为准。

/help查看当前版本支持的命令。
/permissions调整工具权限与确认策略。
/mcp检查 MCP 连接和服务状态。
/agents查看或配置可用子 agent。
/model切换当前会话模型。
/compact压缩上下文，继续长任务。
/clear清空当前上下文。
/init生成或更新项目说明文件。
